Zotero-obsidian for paper note

以前的论文笔记都是在 obsidian 里直接开新的页面记录。最近觉得需要一个表格或者数据库来统一这些笔记，否则记录之后由于难以检索也不会再翻开。obsidian 的表格功能一直做的不好，因此一度想要转向 notion，但是毕竟其他的笔记都在 obsidian 的仓库里，单独把论文笔记拎出去既不合理也不方便。

于是在网上找到了 Zotero + obsidian 的解决方案。正好一并把 ReadPaper 也换成 Zotero，抄袭的总归不如正版好使。

Zotero 安装 Better Bibtex 插件（以及 pdf-translate 插件）。

Obsidian 安装 Zotero Integration 插件，设置如下：

Template 如下：

---
cssclass: paper-note
{% if title %}title: "{{title}}"{% endif %}{% if shortTitle %}
shortTitle: "{{shortTitle}}"{% endif %}
type: "{{itemType}}"{% for type, creators in creators | groupby ("creatorType") -%}{% if loop. first %}
{% endif %}{{type | replace("interviewee", "author") | replace("director", "author") | replace("presenter", "author") | replace("podcaster", "author") | replace("programmer", "author") | replace("cartographer", "author") | replace("inventor", "author") | replace("sponsor", "author")  | replace("performer", "author") | replace("artist", "author")}} : "{%- for creator in creators -%}{%- if creator. name %} {{creator.name}} {%- else %} {{creator.lastName}} , {{creator.firstName}} {%- endif %}{% if not loop. last %}; {% endif %}{% endfor %}"{% if not loop. last %}
{% endif %}{%- endfor %}{% if publicationTitle %}
publication: "{{publicationTitle}}"{% endif %}{% if date %}
date: {{date | format("YYYY-MM-DD")}} {% endif %}{% if archive %}
archive: "{{archive}}"{% endif %}{% if archiveLocation %}
archive-location: "{{archiveLocation}}"{% endif %}
citekey: {{citekey}}
conference:
tags: {% for t in tags -%} {{t.tag | lower | replace (" ", "-")}} {%- endfor %}
takeaway:
---
{{bibliography}}
[online]( {{url}} ) [local]( {{desktopURI}} ) {%- for attachment in attachments | filterby ("path", "endswith", ". pdf") %} [pdf](file:// {{attachment.path | replace(" ", "%20")}} ){% if loop. last %} {% endif %}{%- endfor %}

%image%

注意： 所有的 paper 需要设置 shortTitle，感觉很合理因为本身论文的题目太长了而且经常有冒号之类的奇怪符号。

然后通过 Dataview 来创建一个 paper summary 界面。

```dataview
table title, author, date, archive, conference, tags, takeaway, citekey
from "Papers/Notes"
sort date desc
```

ZeroTier+nomachine 实现更好的远程控制

起因是 TeamViewer 在一阵子之前变成了免费用户只能使用几分钟，纯粹的不干人事。我甚至是在组会的时候发现的，正在连着 lab 的主机给老师展示结果突然连接断开。当时研究半天没明白是怎么回事。

于是找寻替代方案。但大多数远程桌面软件都是要么商业化气息太重，要么质量极差，非常难用。尝试了向日葵，anydesktop，windows远程控制，chrome remote desktop，各有各的坏处。最后找到 nomachine，属于是一个质量尚可、免费、简洁的远程桌面软件。唯一的坏处是只能连接内网机器。这导致家用电脑连接实验室主机需要先连接学校 vpn。

这本身是个很寻常不过的事情。但是今天突然有需求需要从学校电脑连家里的电脑，这就没法先给家里的电脑挂上梯子。于是找到了 ZeroTier 这个软件可以模拟内网环境，并且免费+好用。

不知道为什么设置成 172.26.*.* 的时候就连不上，把内网 ip 格式换成 10.242.*.* 就可以连接。但不是什么大问题。

还发现 nomachine 的 ubuntu 端在非全屏模式下无法监听 windows 键，有网络教程说关掉 wayland 就可以，但是我自己试了不行。全屏模式下就没有这个问题。
