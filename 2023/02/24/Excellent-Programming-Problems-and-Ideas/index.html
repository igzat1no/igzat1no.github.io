Dora and Search·

  • The problem is to find in a permutation of nnn numbers a subsegment such that the leftmost and rightmost elements are not the maximum or the minimum in the subsegment.
  • My brute force idea is, for each element in the permutation, to find the nearest one on the left which is bigger than itself, the nearest one on the left which is smaller, the nearest one on the right which is bigger and the nearest one on the right which is smaller. According to them we can easily determine the left and right boarder lll and rrr for some element aia_iai​ such that aia_iai​ is not the maximum or the minimum in both the range al⋯aia_l \cdots a_ial​⋯ai​ and ai⋯ara_i \cdots a_rai​⋯ar​. Let’s denote the lll and rrr for aia_iai​ as lil_ili​ and rir_iri​.
  • Then, we can iterate lll from 111 to nnn and find t=max⁡k∈[rl,n]lkt = \max_{k \in [r_l, n]} l_kt=maxk∈[rl​,n]​lk​. If t>=lt >= lt>=l, we can make sure that [l,arg⁡max⁡k∈[rl,n]lk]\left[l, \arg \max_{k \in [r_l, n]} l_k \right][l,argmaxk∈[rl​,n]​lk​] is a valid subsegment.
  • The official solution is much simpler. We start from the whole segment [1,n][1, n][1,n], and each step if one of the two ends of the segment does not meet our requirements we can directly remove it. Then we will get some valid solution or there is no valid subsegments.
  • The correctness is quite obvious. If one end does not meet the requirements, for example, the leftmost element is larger than all the other elements, then every subsegment containing the leftmost element (which must start from the left end) should be invalid. So we can directly delete this element and look for our solution in the remaining numbers.
